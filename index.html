專案簡介

本研究旨在透過多種監督式學習模型（例如：決策樹、隨機森林與支持向量機），預測銀行客戶可能流失的風險。 客戶流失一直是金融業的一大挑戰，流失的客戶不僅會直接影響銀行的收入，還可能損害銀行的品牌形象與市場地位。 因此，如何提前預測哪些客戶可能流失並採取有效的挽留措施，是銀行提升客戶關係管理及市場競爭力的重要方向。

銀行客戶流失的主要原因，可能包括服務體驗不佳、競爭對手的吸引、更高的手續費或利率，以及個人財務狀況的變化等。 本專案通過分析多種客戶特徵（如信用評分、年齡、地理位置和產品使用數量等），建構預測模型，協助銀行識別高風險客戶群體， 並提供數據支持以制定精準的留客策略。

我們希望透過本研究，能夠：

  • 提升對客戶資料的洞察力，了解哪些因素最能影響客戶流失。
  • 選擇最佳的機器學習模型，用於準確預測客戶流失的可能性。
  • 為銀行提供可操作的建議，幫助其制定針對性的行銷策略與服務改善方案。

此外，我們也針對模型的準確性、穩定性及應用場景進行詳細比較，力求提供一套兼具實用性與效能的解決方案。

專案介紹

（1）研究背景

在金融市場高度競爭的環境下，銀行面臨客戶流失問題日益嚴重。為提升經營效率與客戶關係管理品質， 透過多種監督式學習技術進行客戶流失預測，能有效協助銀行提前掌握風險，制定精準行銷與留客策略， 強化市場競爭力。

（2）研究問題

  • a. 信用評分、地理位置、性別、年齡等因素對客戶流失率有何影響？
  • b. 哪些因素會對客戶流失率具有最大的影響力？
  • c. 基於這些因素（特徵），要如何預測並降低客戶流失率？

（2）數據集描述

原始資料筆數共有 10002 筆，有 12 個變量，擷取前十筆資料呈現如下：

  • 客戶ID：每個客戶的唯一識別符。
  • 姓氏：客戶的姓氏。
  • 信用評分：代表客戶信用評分的數值。
  • 地理位置：客戶所在的國家（法國、西班牙或德國）。
  • 性別：顧客的性別（男或女）。
  • 年齡：顧客的年齡。
  • 服務年資：客戶在銀行服務的年數。
  • 餘額：客戶的帳戶餘額。

  • HasCrCard：顧客是否有信用卡（1 = 是，0 = 否）。
  • IsActiveMember：客戶是否為活躍會員（1 = 是，0 = 否）。
  • EstimatedSalary：客戶的預期薪資。
  • Exited：客戶是否流失（1 = 是，0 = 否）。

（3）特徵工程

  • Customer ID：識別性質的欄位，並不具備預測價值，因此未納入模型訓練。
  • Surname：僅為識別用途，與客戶是否流失無關，未作為特徵變數。
  • Credit Score：反映顧客信用狀況，是重要的預測變數。
  • Geography：不同地區可能影響流失率，適合作為特徵變數。
  • Gender：性別可能影響流失風險，納入模型訓練。
  • Age：年齡影響流失率，重要的預測變數。
  • Tenure：服務年資反映客戶忠誠度，是重要特徵。
  • Balance：帳戶餘額高低影響流失可能性，作為特徵變數。
  • Number Of Products：使用產品數量與客戶關係密切相關，是重要的預測變數。
  • Has Credit Card：是否持有信用卡，作為客戶與銀行關係的指標之一。
  • Is Active Member：活躍會員對銀行的依賴程度較高，適合作為特徵變數。
  • Estimated Salary：收入水平可能影響流失風險，納入模型訓練。

模型選擇與訓練

（1）採用之模型

在本研究中，我們主要使用了決策樹（Decision Tree）、隨機森林（Random Forest）和支持向量機（SVM）三種分類模型進行預測， 旨在根據各種客戶資料進行客戶流失預測，並選擇最佳模型以達到較高的預測準確度。

  • 決策樹（Decision Tree）：
    • 優點：易於理解與解釋，預測速度快。
    • 缺點：容易過擬合，對資料變動敏感。
  • 隨機森林（Random Forest）：
    • 優點：抗過擬合能力強，適用於高維資料。
    • 缺點：模型較複雜，解釋性較差。
  • 支持向量機（SVM）：
    • 優點：適合處理高維度資料，對小樣本資料也有良好表現。
    • 缺點：參數調整較困難，當資料量很大時，訓練速度慢。
  • 支持向量機（SVM調參後）：
    • 優點：提升分類準確率，能有效改善預設參數下的模型限制，具備良好的泛化能力。
    • 缺點：需要大量計算資源與時間進行參數搜尋，容易因為調整參數過度而造成過擬合。

分析與建議

由上表及混淆矩陣可觀察到，調參後的SVM模型在 Accuracy、Recall 以及 F1 Score 上皆為最高，顯示其對於客戶流失的分類表現最佳； 而決策樹模型在 Precision 指標上表現最高，代表其對於預測「不流失」客戶的精準性較強。隨機森林則在所有評估指標上均表現穩定且為次高， 顯示其兼具準確性與穩定性，是相當均衡的模型選擇。

綜合上述分析，雖然調參後的 SVM 模型在多項指標中表現最佳，但其訓練時間與資源消耗較高，對於即時預測任務來說較不具實務效率。 因此，若考量系統資源與效能表現的平衡，建議可優先採用隨機森林作為客戶流失預測的主要模型，兼顧模型穩定性與實作便利性。

附錄一：Python 程式碼（決策樹）

        import pandas as pd
        import numpy as np
        import matplotlib.pyplot as plt
        import seaborn as sns
        # scikit-learn 相關匯入
        from sklearn.model_selection import train_test_split
        from sklearn.tree import DecisionTreeClassifier
        from sklearn.metrics import accuracy_score, classification_report, confusion_matrix
        # 1. 讀取 CSV 檔案
        data = pd.read_csv('Churn_Modelling.csv')
        data = data.drop(['RowNumber', 'CustomerId', 'Surname'], axis=1)
        # 2. 缺失值填補
        data['Geography'] = data['Geography'].fillna(data['Geography'].mode()[0])
        data['Age'] = data['Age'].fillna(data['Age'].median())
        # 3. One-Hot Encoding for Geography
        from sklearn.preprocessing import OneHotEncoder
        ohe = OneHotEncoder()
        geo_encoded = ohe.fit_transform(data[['Geography']]).toarray()
        geo_encoded_df = pd.DataFrame(geo_encoded, columns=ohe.get_feature_names_out(['Geography']))
        data = pd.concat([data, geo_encoded_df], axis=1)
        data = data.drop('Geography', axis=1)
        # 4. 模型建立與評估
        X = data.drop(['Exited'], axis=1)
        y = data['Exited']
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.1, random_state=42)
        dt_classifier = DecisionTreeClassifier(max_depth=5, random_state=42)
        dt_classifier.fit(X_train, y_train)
        y_pred = dt_classifier.predict(X_test)
        print("Accuracy:", accuracy_score(y_test, y_pred))
        print("Confusion Matrix:\n", confusion_matrix(y_test, y_pred))

        # 1. 讀取資料
        df = pd.read_csv('Churn_Modelling.csv')
        df.drop(['RowNumber', 'CustomerId', 'Surname'], axis=1, inplace=True)
        # 2. 資料處理與特徵工程
        from sklearn.model_selection import train_test_split
        from sklearn.preprocessing import LabelEncoder, OneHotEncoder
        le = LabelEncoder()
        df['Gender'] = le.fit_transform(df['Gender'])
        ohe = OneHotEncoder()
        geo_encoded = ohe.fit_transform(df[['Geography']])
        geo_encoded_df = pd.DataFrame(geo_encoded, columns=ohe.get_feature_names_out(['Geography']))
        df = pd.concat([df.drop('Geography', axis=1), geo_encoded_df], axis=1)
        X = df.drop('Exited', axis=1)
        y = df['Exited']
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.1, random_state=42)
        # 3. 模型建立與訓練
        rf_classifier = RandomForestClassifier(n_estimators=100, random_state=42)
        rf_classifier.fit(X_train, y_train)
        y_pred = rf_classifier.predict(X_test)
        print("Accuracy:", accuracy_score(y_test, y_pred))
